【解决】Ubuntu环境下更换网络下载源

准备做一下线性规划的模型发现需要用到pulp，本来是挺简单的

pip install pulp

就是这么简单的一个命令，直接提示“Defaulting to user installation because normal site-packages is not writeable”，显示不可写，后来用了sudo的管理员命令能写是有权限能写了，但发现下载了0.1mb的时候就停止了，我就觉得是pip工具的网络源问题。

遇到这个问题的小伙伴可以修改一下网络源，换一个清华的镜像源（这里展示的是永久修改pip源）。

  • 创建pip.conf的文件
mkdir ~/.pip # 如果有这个文件夹的不需要建立
cd ~/.pip # 转到.pip这个文件夹
touch pip.conf # 建立这个文件
sudo vi ~/.pip/pip.conf # 用vim编辑器编辑这个文件
  • 接下来用vim编辑器编辑pip.conf这个文件
[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple # 改为清华镜像源
[install]
trusted-host = https://pypi.tuna.tsinghua.edu.cn # 此参数为了保证受信任，避免麻烦
  • 接下来保存好退出vim就好，然后再使用最初那个pip的下载命令就没关系了
